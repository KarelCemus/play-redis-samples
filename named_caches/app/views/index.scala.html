Named Caches example!

This application shows the configuration of multiple named caches and custom recovery policy with play-redis.

All messages are cached for 10 seconds. All messages are cached under the same key.

First, there is a default unqualified cache bound to local with the current message: @default

Second, there is a local cache. The current message is @local

Third, there is a remote cache. The current message is @remote

Finally, there is a failing cache pointed to the dead instance. It uses a custom recovery policy and the cache is disabled, thus there is no caching. The current message is @failing

Right now, it's @now.
